# Deadline 雷达开发者预览版使用教程

本文档面向想要本地运行和试用 Deadline 雷达的开发者。当前版本是 Developer Preview / MVP，不提供安装包，需要通过源码启动 Electron 桌面应用。

## 当前状态

Deadline 雷达当前已经打通：

- 粘贴文本解析 deadline。
- 飞书 Doc/Docx 链接直读解析。
- 风险雷达和事项队列展示。
- 本地 JSON 文件存储。
- 通过本机 `lark-cli --as user` 同步到飞书日历。

当前还没有：

- macOS `.dmg` 或 Windows `.exe` 安装包。
- 应用签名、自动更新。
- API Key 加密持久化。
- 内置飞书 OAuth，仍依赖本机 `lark-cli`。

## 使用前提

本地解析和飞书同步的前提不同。只想体验文本解析时，可以先不配置飞书同步。

### 必须准备

- Node.js
- npm
- 一个 OpenAI-compatible 模型服务
  - API Base URL
  - API Key
  - Model 名称

### 同步飞书日历还需要

- 已安装 `lark-cli`。
- 当前飞书账号可以使用日历。
- `lark-cli doctor` 可以正常运行。
- `lark-cli --as user` 已完成授权，并具备读取、创建日历和创建日程的权限。

可先在终端检查：

```sh
lark-cli doctor
lark-cli calendar calendars list --as user --params '{"page_size":50}'
```

如果 Calendar 权限未就绪，可以在应用设置页点击“启动授权”，或在终端执行：

```sh
lark-cli auth login --domain calendar
```

## 获取代码

```sh
git clone https://github.com/sparkler233/deadline-radar.git
cd deadline-radar
npm install
```

## 启动应用

开发者预览版推荐用 Electron 开发模式启动：

```sh
npm run dev
```

启动后会打开 Electron 桌面窗口。请在这个桌面窗口里测试飞书 CLI 检测、授权、文档读取和日历同步。

不要直接把 `http://127.0.0.1:5173/` 当正式应用使用。普通浏览器打开时只会进入前端预览模式，不能调用本机 `lark-cli`。

## 第一次配置

打开应用后进入“设置”页。

### 模型配置

填写：

- `API Base URL`
- `API Key`
- `Model`
- `Temperature`
- `请求超时 ms`

示例：

```text
API Base URL: https://api.deepseek.com
Model: deepseek-v4-flash
Temperature: 0.2
请求超时 ms: 45000
```

注意：当前 MVP 不会把 API Key 写入本地 JSON。API Key 只保存在当前运行会话里，应用重启后需要重新填写。

填写完成后点击“保存设置”。

### 飞书配置

填写或确认：

- `lark-cli 路径`
  - 默认可用时填 `lark-cli` 即可。
  - 如果系统 PATH 找不到，可以填绝对路径。
- `专用日历名称`
  - 默认是 `Deadline 雷达`。
- `默认时区`
  - 默认是 `Asia/Shanghai`。

点击“检测飞书 CLI”。看到 `CLI 已找到`、`doctor 通过`、`user 日历可用` 后，就可以同步日程。

如果提示需要授权，点击“启动授权”，按打开的飞书页面完成确认。应用会等待 `lark-cli` 写入 user token，然后再次检测权限。

## 解析一段通知文本

进入“风险雷达”页，把通知正文粘贴到大文本框。

示例：

```text
@所有人 明天路演的同学注意一下哈

1. 今晚 21:30 前，每个小组把最终版 PPT 发到群里，文件名统一写成“组号-项目名-路演PPT”。
2. 明天上午 10:00 前，组长把 1 分钟项目介绍词发给我，我这边要统一排版。
3. 路演正式开始时间是 2026 年 6 月 4 日 14:30，地点暂定创新楼 302，大家最好 14:10 前到场调试设备。
4. 还没交报名表的同学，今天晚上之前私聊我补一下，不然明天签到可能对不上。
5. 最近两天大家记得检查一下数据来源和图片版权，老师可能会问。

收到请回复 1，辛苦大家～
```

来源标题可以填写：

```text
微信群路演通知
```

点击“解析事项”。解析成功后，应用会生成结构化事项，并展示在风险雷达和事项队列中。

## 解析飞书文档链接

当前版本支持飞书 Doc/Docx 链接直读。

使用方式：

1. 确保飞书 CLI 检测通过。
2. 在“风险雷达”页保持正文输入框为空。
3. 在“来源链接”里填入飞书文档链接。
4. 点击“解析事项”。

应用会通过下面的方式读取文档正文：

```sh
lark-cli docs +fetch --api-version v1 --as user --doc <飞书文档链接>
```

读取成功后，再把正文交给模型解析。

如果文档读取失败，常见原因是：

- 当前飞书账号没有这个文档的访问权限。
- `lark-cli --as user` 没有完成授权。
- 链接不是飞书 Doc/Docx。

这种情况下，可以先手动复制文档正文，粘贴到文本框里解析。

## 管理事项

进入“事项队列”页，可以对解析出的事项做这些操作：

- 确认：表示这个事项有效，准备进入后续处理。
- 完成：表示事项已经处理完。完成事项会保留在全部事项里，但不会进入风险雷达和同步队列。
- 删除：从当前可见列表中移除。
- 同步：把事项写入飞书日历。

没有明确时间的事项会显示为不可推送，需要先补充时间后才能同步。当前版本还没有内置编辑时间能力，这是后续计划。

## 同步到飞书日历

点击事项卡片上的“同步”按钮后，应用会：

1. 查找是否已经存在 `Deadline 雷达` 专用日历。
2. 如果不存在，则创建这个专用日历。
3. 在专用日历中创建日程。
4. 把同步结果写回本地 `items.json`。

同步成功后，事项会显示为“已同步”。飞书 App 会继续负责日历提醒和跨设备查看。

## 本地数据

Electron 模式下，数据保存在系统应用数据目录中。

主要文件：

- `settings.json`
  - 模型非敏感配置、飞书配置、UI 偏好。
- `items.json`
  - 解析出的事项列表。
- `sync-queue.json`
  - 待同步和失败重试事项。

API Key 不会写入这些 JSON 文件。

## 常见问题

### 页面提示缺少 API Key

进入设置页重新填写 API Key。当前版本重启后需要重新填写。

### 浏览器里不能检测飞书 CLI

这是正常的。普通浏览器预览模式不能调用本机 `lark-cli`。请使用 `npm run dev` 打开的 Electron 桌面窗口。

### 检测飞书 CLI 失败

检查：

```sh
lark-cli doctor
```

如果终端可以运行，但应用检测不到，可以在设置页把 `lark-cli 路径` 改成绝对路径。

### 授权后仍提示没有 Calendar 权限

请在应用里重新点击“检测飞书 CLI”。如果仍失败，可以重新点击“启动授权”。

当前应用内授权流程会先打开飞书授权页面，再等待 `lark-cli auth login --device-code ...` 写入 user token。

### 飞书文档链接读取失败

优先确认：

- 当前飞书账号是否能打开这个文档。
- `lark-cli docs +fetch --api-version v1 --as user --doc <链接>` 是否能在终端读取正文。

如果不能读取，请先复制正文到文本框里解析。

### 日历同步失败

优先确认：

```sh
lark-cli calendar calendars list --as user --params '{"page_size":50}'
```

如果这个命令失败，说明 user 日历权限还没准备好。

```sh
npm run build
```

生产依赖安全检查：

```sh
npm audit --omit=dev
```

## 当前限制

- 没有安装包，需要源码运行。
- API Key 只保存在运行时内存。
- 飞书授权依赖本机 `lark-cli`。
- 已同步事项暂未实现更新已有日程，只完成创建路径。
- 无明确时间事项暂时不能在应用内编辑时间。
- 重复同步同一事项的幂等行为仍需进一步打磨和测试。
